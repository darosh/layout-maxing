# layout-maxing

> A Max patch layout utility

With the magick of genetic algorithm:
- it may reorganize your simple patch in way you won't like, but it should not take much time
- it may turn your big messy patch into a different kind of mess in a hour or so, it depends on your CPU core count 

[Live demo](https://darosh.github.io/layout-maxing)

## Development (Vite+ Monorepo)

- Check everything is ready:

```bash
vp run ready
```

- Run the tests:

```bash
vp run -r test
```

- Build the monorepo:

```bash
vp run -r build
```

- Run the development server:

```bash
vp run dev
```
